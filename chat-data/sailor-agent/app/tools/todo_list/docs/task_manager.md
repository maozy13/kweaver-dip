#  任务管理工具
管理任务执行的顺序，状态等



## 获取当前会话可执行任务
1. 根据会话ID获取任务列表
2. 根据依赖顺序，返回可以执行的任务

什么可以做? -- 状态为 pending 且 blockedBy 为空的任务。
什么被卡住? -- 等待前置任务完成的任务。
什么做完了? -- 状态为 completed 的任务, 完成时自动解锁后续任务。



## 更新任务的状态
### 参数
```
adjust: true
reason: 需要调整的原因
```

### 逻辑
1. 更新任务的结果，完成，失败等
2. 如果传入参数说明需要调整任务结果，那就根据任务重新生成任务列表
3. 可能整个问题在某一步已经可以确认全部完成了，就会删除剩余的执行步骤。
4. 如果最后一个任务已经完成，删除整个任务列表
